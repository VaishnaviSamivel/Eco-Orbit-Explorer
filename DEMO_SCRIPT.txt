[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at how this system operates behind the scenes.
First, every user prompt starts at the Security Checkpoint node, which scrubs PII, blocks prompt injections, and scans for illegal activity keywords.
If clean, the request flows to our main Orchestrator Agent.
The Orchestrator identifies the user's core intent and delegates the heavy lifting to one of our specialized sub-agents: the Eco Data Agent for satellite and air indices, or the Sustainability Agent for emission calculations.
These sub-agents connect directly to our local FastMCP server, executing specific stdio tools like calculate_carbon_footprint or get_deforestation_data.
For high-impact decisions, such as filing regulatory alerts, the Orchestrator routes the flow through a Human Review Node, pausing execution until a user manually approves the action.
Finally, the results are unified and output through the Final Output Node.
